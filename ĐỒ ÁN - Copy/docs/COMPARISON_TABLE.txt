# SO SÁNH 3 THUẬT TOÁN SEARCH THUẦN TÚY

```
╔═══════════════════════════════════════════════════════════════════════════╗
║                   3 THUẬT TOÁN SEARCH THUẦN TÚY                           ║
╚═══════════════════════════════════════════════════════════════════════════╝

┌─────────────────────────────────────────────────────────────────────────┐
│ 1. GREEDY BEST-FIRST SEARCH (GBFS)                                     │
├─────────────────────────────────────────────────────────────────────────┤
│ Công thức:     Chọn node có h(n) nhỏ nhất                              │
│ Sử dụng:       h(n) = distance[city][goal]                             │
│ Không dùng:    g(n)                                                     │
│                                                                         │
│ Ví dụ bước chọn:                                                        │
│   Candidate A: h=100 km ← CHỌN (gần goal nhất)                         │
│   Candidate B: h=150 km                                                 │
│   Candidate C: h=200 km                                                 │
│                                                                         │
│ ✅ Ưu điểm:    Nhanh, ít operations                                     │
│ ❌ Nhược điểm: Không tối ưu, tham lam                                   │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│ 2. UNIFORM COST SEARCH (UCS)                                           │
├─────────────────────────────────────────────────────────────────────────┤
│ Công thức:     Chọn node có g(n) nhỏ nhất                              │
│ Sử dụng:       g(n) = total_distance + distance[current][city]         │
│ Không dùng:    h(n)                                                     │
│                                                                         │
│ Ví dụ bước chọn:                                                        │
│   Candidate A: g=250 km (tích lũy)                                      │
│   Candidate B: g=200 km ← CHỌN (chi phí tích lũy thấp nhất)            │
│   Candidate C: g=300 km                                                 │
│                                                                         │
│ ✅ Ưu điểm:    Tối ưu (optimal), đầy đủ (complete)                      │
│ ❌ Nhược điểm: Chậm hơn, nhiều nodes explored                           │
└─────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────┐
│ 3. A* ALGORITHM                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│ Công thức:     Chọn node có f(n) = g(n) + h(n) nhỏ nhất                │
│ Sử dụng:       g(n) = total_distance + distance[current][city]         │
│                h(n) = distance[city][goal]                              │
│                f(n) = g(n) + h(n)                                       │
│                                                                         │
│ Ví dụ bước chọn:                                                        │
│   Candidate A: g=250, h=100, f=350                                      │
│   Candidate B: g=200, h=150, f=350                                      │
│   Candidate C: g=150, h=180, f=330 ← CHỌN (f nhỏ nhất)                 │
│                                                                         │
│ ✅ Ưu điểm:    Tối ưu, nhanh, cân bằng                                  │
│ ❌ Nhược điểm: Nhiều operations nhất (tính cả g và h)                   │
└─────────────────────────────────────────────────────────────────────────┘

╔═══════════════════════════════════════════════════════════════════════════╗
║                        BẢNG SO SÁNH CHI TIẾT                              ║
╚═══════════════════════════════════════════════════════════════════════════╝

┌─────────────────┬──────────────┬──────────────┬──────────────────────────┐
│     Tiêu chí    │  Greedy BFS  │     UCS      │          A*              │
├─────────────────┼──────────────┼──────────────┼──────────────────────────┤
│ Sử dụng         │   h(n) only  │  g(n) only   │   f(n) = g(n) + h(n)     │
├─────────────────┼──────────────┼──────────────┼──────────────────────────┤
│ Tối ưu          │      ❌      │      ✅      │      ✅ (h admissible)    │
├─────────────────┼──────────────┼──────────────┼──────────────────────────┤
│ Đầy đủ          │      ❌      │      ✅      │          ✅              │
├─────────────────┼──────────────┼──────────────┼──────────────────────────┤
│ Tốc độ          │   ⚡ Nhanh   │   🐌 Chậm    │      ⚡ Nhanh            │
├─────────────────┼──────────────┼──────────────┼──────────────────────────┤
│ Operations      │   30 (ít)    │   34 (TB)    │     54 (nhiều)           │
│ (5 cities)      │              │              │                          │
├─────────────────┼──────────────┼──────────────┼──────────────────────────┤
│ Nodes explored  │      10      │      10      │          10              │
│ (TSP 5 cities)  │              │              │                          │
├─────────────────┼──────────────┼──────────────┼──────────────────────────┤
│ Độ phức tạp     │   O(b^m)     │  O(b^C*/ε)   │       O(b^d)             │
│ thời gian       │              │              │                          │
├─────────────────┼──────────────┼──────────────┼──────────────────────────┤
│ Độ phức tạp     │   O(b*m)     │  O(b^C*/ε)   │       O(b^d)             │
│ không gian      │              │              │                          │
├─────────────────┼──────────────┼──────────────┼──────────────────────────┤
│ Phù hợp cho     │ Tìm nhanh    │ Cần tối ưu   │  Cân bằng tốc độ        │
│                 │ xấp xỉ       │ chi phí      │      & tối ưu            │
└─────────────────┴──────────────┴──────────────┴──────────────────────────┘

Chú thích:
  b = branching factor (số nhánh trung bình)
  m = độ sâu tối đa
  d = độ sâu nghiệm
  C* = chi phí optimal
  ε = bước chi phí nhỏ nhất
```
